# 蜂巢奇遇

一个可以直接运行的手机端单人小游戏，画面风格是明亮二次元奇幻冒险：

- `index.html`：手机浏览器版，适合发给朋友试玩。
- `stardash.py`：旧的 Python + tkinter 桌面版，保留在目录里。

## 手机玩法

最简单的玩法是：朋友点一个网页链接，直接进游戏。

这个游戏不需要安装 App，不需要注册账号，不需要下载文件。只要你把这 3 个文件放到一个能访问的网址里：

1. `index.html`
2. `styles.css`
3. `game.js`

然后把网址发给朋友，朋友手机点开就能玩。

## 操作

- 按住屏幕任意位置拖动男性小蜜蜂
- 收集 10 种发光仙姬，躲开 10 种古怪女妖
- 顶部显示分数、生命、等级、最高分

## 关于发给朋友

不要直接发本地文件给朋友。手机收到 `.html` 文件后，经常会打不开，或者样式和游戏逻辑丢失。

正确方式是：发网页链接。

如果 `https://app.netlify.com/drop` 打不开或打开慢，不要继续卡在那里。这个游戏是纯静态网页，可以换任意静态托管：

- 国内朋友试玩：优先用国内服务器、对象存储静态网站、腾讯云 EdgeOne Pages、阿里云 OSS 静态网站这类入口。
- 海外访问：GitHub Pages、Vercel、Cloudflare Pages、Netlify 都可以。
- 临时同 Wi-Fi 测试：VS Code 安装 Live Server，电脑开服务后，手机访问电脑局域网 IP。

后面要做真正的“多人联机”，需要服务器或 WebSocket 房间系统；单个静态网页更适合先做单人玩法验证。

## 性能策略

手机版已经做了几项基础优化：

- 限制高倍屏渲染倍率，避免 3 倍屏把画布渲染压力拉满。
- 静态背景只绘制一次，运行时复用缓存。
- 限制星光、女妖、仙姬、粒子数量。
- HUD 只在数值变化时刷新。
- 页面切到后台时不推进游戏逻辑。
